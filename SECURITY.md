# Security Policy

## Overview

Security is a top priority for **HyperStreamKit**.

As a next-generation Android streaming SDK, HyperStreamKit is designed with
performance, reliability, and security in mind.

This project is developed and maintained by **cybernahid-dev**.

---

## Supported Versions

Only the latest stable release of HyperStreamKit is actively supported with
security updates.

| Version | Supported |
|--------|-----------|
| Latest | ✅ Yes |
| Older | ❌ No  |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

❗ **Do NOT open a public GitHub issue for security vulnerabilities.**

Instead:

- Clearly describe the vulnerability
- Include steps to reproduce (if possible)
- Explain potential impact
- Share logs or proof-of-concept responsibly

You may contact the maintainer through private communication channels or
GitHub security advisories.

---

## Responsible Disclosure

We follow responsible disclosure practices:

- Vulnerabilities will be investigated promptly
- Fixes will be prioritized based on severity
- Public disclosure will occur only after a fix is available

We appreciate researchers and contributors who help keep the project secure.

---

## Security Best Practices

Contributors are encouraged to:

- Avoid introducing unsafe native code
- Validate all external inputs
- Handle network data securely
- Minimize attack surface in public APIs
- Follow Android security best practices

---

The maintainers are not responsible for misuse, misconfiguration, or
third-party integrations.

## Final Note

Security is a shared responsibility.

If you see something suspicious, say something.

Thank you for helping keep HyperStreamKit safe.
